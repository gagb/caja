    Welcome!

    This task is part of a research study investigating human/AI collaboration. In it, you'll be playing a game with help from Marvin the Robot

    You will be presented with a variety of objects, one-by-one. You (with Marvin helping) are required to label these objects as "DEFECTIVE" or "NOT-DEFECTIVE".

    For example, the screenshot below contains a small, blue, circle-shaped object. A table next to the object enumerates the various features that make up the object and their values.

    The Task

    We would like you to check to see if a visual object presented to you on the screen is defective. The process of checking to see if an object is defective is complicated, with many variables that might be hidden from you (i.e., not shown in the visual object or in the feature list provided).

    Marvin the robot has learned how to identify defects using all the dimensions of an object, even those you can’t see, but only 80% of the time. Therefore, Marvin's advice to you is going to be wrong for some of the objects around 20% of the time.

    Your task is to figure out if it is safe to trust Marvin for an object that is shown to you based on what you can see. The objects for which Marvin makes mistakes may be correlated with the features presented to you. Learning these patterns should help you form a strategy so that you can earn more bonus.

    Game controls and scoring

    Whenever you encounter an object, you have the option to label it either using Marvin's help or use the 'COMPUTE' button.

    • If you use Marvin and his suggestion is correct, you will earn $.
    • If you use Marvin and his suggestion is wrong, you will lose $.
    • If you click on "COMPUTE" you are guaranteed to submit the correct label but earn $, because of the extra computational overhead.
